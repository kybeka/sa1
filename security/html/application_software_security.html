    Software Security vs.
    Application security

    To safeguard software from malware, there are two layers of defense to keep in mind, external and internal. The former method concerns application security while the latter involves software security. It is helpful to compare application security to the outer walls of a castle and software security to the army inside the castle.




    The outer walls

    Application security concerns the use of post-development methods to protect the software, this can take on many forms such as filtering the inputs for software by detecting and blocking malicious code and trying to patch (fix a problem after being detected) the software after its release. In this case, other programs might be used to ensure protection such as antivirus software or even hardware.
    For example, in the case of software that are connected to the internet, blocking certain types of traffics (data carried over a network) that have the potential to cause harm is considered a kind of application security.

    The army

    Software security focuses on securing the software by focusing on its developpment stage. In other words software Security is about designing the code of the software while keeping security in mind, limiting the number of vulnerabilities and reducing the attack surface (i.e. different points from which an attack can take place).
    If we take a closer look at the software development life cycle (the process with which a program is developped),
    we can see that security can play a role at each stage. For instance, when designing the product, checking for
    vulnerabilities in third-party software component(parts of the software that are made by a third party) should be
    a priority, furthermore when testing the product before publication, the developpers should take into consideration not only the performance of the software but also the result of the security tests.
