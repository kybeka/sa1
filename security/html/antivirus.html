    In this article, we will examine the different methods deployed by these software to counter the different types of malware.

    Signature Based Detection

    When trying to detect malware, the first line of defense for most antivirus software is black listing the biggest possible number of malware. This procedure is made possible by malware analysts that are hired by the company owning the antimalware software. Generally speaking, dedicating more manpower and resources to this procedure makes the blacklist more robust. For the majority of malware, each one of them has a specific fingerprint or signautre that can be used to identify it just like in the case of a real virus.
    Once a virus is encountered by a software analyst its pattern is extracted and then recorded in the blacklist and because most of the malware that exist are modifications of other malware, they can be grouped into families that share the same or similar fingerprints, this is specifically true in the case of computer viruses.
    Once a malware is detected by the software, the program or file is "quarantined", the malware is isolated and can no longer harm the system by replicating or infecting and modifying other files.

    Behavior Based Detection

    For the most part, Signature based detection fends off the majority of viruses and other malware such as ransomware. Nevertheless, this method is helpless against new malware that are not encountered before and against polymorphic viruses (viruses that modify part of their code each time they run).In this case the antivirus software might use a second more "intelligent" line of defense. Behavior based detection method uses statistics and machine learning to detect deviations from a certain known malware fingerprint, it can also scan the code line by line and try to predict the potential actions of the code. To do so, the software might use a virtual machine to test the suspected program, in other words, the software simulates the execution of the suspected file in a closed safe environment to detect any malicious behavior such as requesting access to critical information or attempting to encrypt files. Nevertheless, this method also has its drawbacks it can slow down the computer performance and it can be bypassed by the malware in certain cases. For instance certain malware such as trojan malware can be hidden inside a legitimate program and might not exhibit malicious behavior until after a certain time delay or only when they are not inside a virtual machine.
